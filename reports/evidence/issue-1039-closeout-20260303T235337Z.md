# Issue #1039 – Closeout Evidence

## Summary

Mobile horizontal overflow on `/gui` at `360x800` is fixed by tightening responsive width constraints in the main layout and switching map-legend rows from wrapping flex to bounded grid columns.

## Implemented

- `src/shared/gui_mvp.py`
  - `main > section { min-width: 0; }` added (prevents grid items from forcing oversized min-content width).
  - `.card` + `.map-shell` + `.map-surface` now explicitly allow shrink (`min-width: 0`).
  - `.map-legend` changed from wrapping flex to grid (`repeat(2, minmax(0, 1fr))`) with narrow-breakpoint fallback to one column.
  - `.map-tile-layer` now clips overflow explicitly.
- `scripts/run_issue_1039_mobile_overflow_smoke.cjs`
  - New E2E smoke runner with direct assertion:
    - `document.scrollingElement.scrollWidth === document.scrollingElement.clientWidth`
  - Includes mobile (`360x800`) + desktop (`1280x800`) checks.
- `docs/testing/GUI_MOBILE_OVERFLOW_SMOKE.md`
  - Added execution guide, assertion, artifact contract, and desktop regression note.
- `tests/test_gui_mobile_overflow_smoke_docs.py`
  - Added doc/script guard tests for issue #1039 smoke documentation.
- `tests/test_web_service_gui_mvp.py`
  - Updated CSS contract checks for new responsive/grid rules.
